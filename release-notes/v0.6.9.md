# pi-pretty 0.6.9

## Summary

This patch release fixes the published extension crash on Pi session start/resume caused by a missing runtime dependency declaration.

## What changed

- Added `typebox` as a direct runtime dependency.
- Bumped package metadata to `0.6.9`.
- Refreshed `package-lock.json` so clean installs match the published package metadata.

## Why

`src/tools/multi-grep.ts` imports `Type` from `typebox` at top level. The built `dist/tools/multi-grep.js` therefore runs `require("typebox")` while Pi loads the extension.

Local development installs masked this because `typebox` was present through the workspace dependency tree. A clean published extension install did not guarantee that module, so Pi could crash before the extension reached its `session_start` handlers.
