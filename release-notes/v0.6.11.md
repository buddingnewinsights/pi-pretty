# pi-pretty 0.6.11

## Summary

Fixes the Pi resume crash seen with `@heyhuynhgiabuu/pi-pretty@0.6.10` on Pi `0.79.9`.

## What changed

- Updated the runtime `@earendil-works/pi-coding-agent` dependency from `^0.75.1` to `^0.79.9`.
- Added `@earendil-works/pi-ai` as a dev dependency for current type-checking.
- Bumped package metadata to `0.6.11`.
- Refreshed `package-lock.json`.

## Why

`v0.6.10` made `@earendil-works/pi-coding-agent` a runtime dependency so the published package could register pretty wrappers and FFF commands. But it used the old `^0.75.1` SDK range. On Pi `0.79.9`, that meant the extension loaded an older nested Pi SDK/runtime while the host Pi process was running `0.79.9`, which could crash during session resume.

`v0.6.11` aligns the published extension dependency with Pi `0.79.9`.
